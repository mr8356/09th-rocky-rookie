## 네트워크 기본 개념과 인터페이스 구성 이해

### 1. 네트워크(Network)

**여러 장치들이 서로 연결되어 데이터를 주고받을 수 있도록 만든 구조**  
-> 내 컴퓨터가 다른 컴퓨터와 어떻게 연결되고, 어떤 경로로 데이터를 주고받는가를 다룸

<pre>
내 컴퓨터(또는 서버)
   │
   │  네트워크 인터페이스
   │
IP 주소 부여
   │
같은 네트워크와 통신
   │
게이트웨이를 통해 외부 네트워크로 나감
   │
DNS를 통해 이름을 IP로 변환
   │
인터넷 / 다른 서버와 통신
</pre>

### 2. IP 주소

**네트워크에서 장치를 식별하기 위한 주소**  
예를 들어 어떤 컴퓨터가 다른 컴퓨터에게 데이터를 보내려면 누구에게 보내야 하는지 알아야 한다. 그 역할을 하는 것이 IP 주소다.

#### 2-1. IPv4 주소의 구조

<pre>192.168.10.15</pre>

- 이 주소는 4개의 숫자로 이루어져 있고, 각 숫자는 점(.)으로 구분된다.
- 각 칸을 옥텟(octet)이라고 부르며, 각 옥텟은 0~255 범위를 가진다.
- 총 32비트의 주소 (약 43억 개)

#### 2-2. CIDR

IP 주소의 network prefix가 어디까지인지 알기 위해 명시하는 마스크 값

<pre>192.168.10.15/24</pre>

여기서 CIDR 값은 `24`이다  
-> network prefix: 24bit
=> 192.168.10.X끼리는 같은 network 범위 안에 있다!

#### 2-3. 공인 IP와 사설 IP

- **공인 IP (Public IP)**: 인터넷에서 직접 사용되는 주소
  - ISP(인터넷 서비스 제공자)가 할당 및 분배
- **사설 IP (Private IP)**: 내부 네트워크에서만 사용하는 주소
  - 회사 내부망, 집 공유기 네트워크, 가상머신 네트워크 등에서 많이 사용함
  - **why?** 인터넷에 연결되는 모든 장치에 공인 IP를 하나씩 주기 어렵기 때문이다. 그래서 내부에서는 사설 IP를 쓰고, 외부 인터넷과 통신할 때는 보통 공유기나 라우터가 NAT를 통해 공인 IP로 변환해준다.

### 3. 게이트웨이(Gateway)

**다른 네트워크로 나갈 때 거치는 출입구**
내 컴퓨터가 같은 네트워크 안의 장치와 통신할 때는 직접 보낼 수 있다. 하지만 다른 네트워크나 인터넷으로 나가려면,
보통 라우터 또는 공유기를 통해 나가야 한다. 그 라우터의 주소가 게이트웨이다.

### 4. DNS(Domain Name System)

**도메인 이름과 IP 주소를 서로 변환하는 역할**  
사람은 `google.com`, `naver.com`처럼 이름을 기억하기 쉽다. 하지만 컴퓨터는 실제 통신할 때 IP 주소가 필요하다. 그래서 DNS가 중간에서 이름을 IP 주소로 바꿔준다.

<pre>google.com → 142.x.x.x</pre>

1. 설정된 네임 서버로 DNS 요청이 들어온다.
2. 전송된 도메인 이름을 DNS 정보가 모두 저장된 데이터베이스인 DNS 쿼리와 비교한다.
3. 해당 도메인에 맞는 IP 주소를 사용자에게 반송하여 접속이 이뤄지게 만든다.

### 5. 네트워크 인터페이스(Network Interface)

**컴퓨터가 네트워크에 연결되도록 도와주는 매개체**
실제로 물리적 실체를 가질 수도 있고, 소프트웨어만으로도 구현될 수 있다.

#### 5-1. 주요 역할

- 데이터 송수신: 데이터를 패킷 형태로 변환하여 데이터 전송 및 처리
- 주소 관리: 장치 고유 식별자인 MAC 주소를 통해 네트워크 상의 다른 장치와 통신
- 프로토콜 처리: TCP/IP 같은 프로토콜을 통해 데이터 전송 관리

#### 5-2. NIC(Network Interface Card)

- 컴퓨터에 전용 네트워크를 제공하는 회로 기판
- 컴퓨터의 메인보드에 기본적으로 내장되어 있음 (LAN card로도 불림)
- 종류: Ethernet NIC, Wireless NIC

#### 5-3. Ethernet NIC

Ethernet NIC는 해당 카드에 케이블을 연결해야 함

#### 5-4. Wireless NIC

케이블 연결이 필요하지 않은 무선 통신을 지원하며 카드에 통합된 작은 안테나로 구성됨
ex) Wi-Fi, Bluetooth

#### 5-5. Virtual Network Interface

- 실제 랜카드가 아닌, OS나 가상화 환경에서 생성되는 논리적 인터페이스
- 가상머신, 컨테이너, 브리지 네트워크 등에서 사용됨
- 하나의 물리 인터페이스 위에 여러 개의 가상 인터페이스를 만들 수 있음

#### 5-6. Loopback Network Interface

자기 자신을 가리키는 역할을 하는 가상의 인터페이스

- 외부 네트워크를 거치지 않고 자기 자신과 통신할 때 사용됨

특징

- 네트워크 스택이 정상적으로 동작하는지 테스트 가능
- 실제 네트워크 연결 없이도 서버 내부 통신 가능
- 항상 활성화되어 있음 (DOWN 상태 거의 없음)

ex)

- 로컬 서버 테스트 (`localhost`)
- 내부 서비스 간 통신
- DB, 웹 서버 로컬 연결

#### 5-7. MAC 주소

- 네트워크 인터페이스의 고유 식별자
- 장치 제조시 할당 (주민등록번호 개념)
- 데이터 송수신 시 장치를 구분하는 데 사용됨

역할

- 같은 네트워크 내에서 실제 데이터 전달 대상 식별
- ARP(Address Resolution Protocol)를 통해 IP → MAC 주소 변환
